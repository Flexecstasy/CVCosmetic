# CVCosmetic — ML-пайплайн распознавания этикеток косметики

Система выполняет OCR косметических этикеток (русский + английский), извлекает структурированные данные о продукте и предоставляет REST API для интеграции.

---

## Архитектура

```
Изображение (JPG/PNG)
        │
        ▼
  ┌─────────────┐
  │  EasyOCR    │  ← модели ru + en, CPU-only
  │ (ocr.py)    │
  └──────┬──────┘
         │ raw text
         ▼
  ┌─────────────────┐
  │  Нормализация   │  ← unicode NFC, схлопывание пробелов,
  │ (normalizer.py) │    удаление управляющих символов
  └──────┬──────────┘
         │
         ▼
  ┌─────────────────┐
  │   Извлечение    │  ← бренд, тип, объём, штрихкод,
  │  (extractor.py) │    сырой состав
  └──────┬──────────┘
         │
         ▼
  ┌─────────────────┐
  │  Парсинг INCI   │  ← разбиение по , ; •
  │ (normalizer.py) │    удаление русских пояснений
  └──────┬──────────┘
         │
         ▼
   JSON-ответ API
```

### Структура metadata.json

```json
{
  "url": "https://...",
  "title": "Аква-крем для лица `ЧИСТАЯ ЛИНИЯ` ИДЕАЛЬНАЯ КОЖА 50 мл",
  "composition": "Aqua, Glycerin, Panthenol, Niacinamide, ...",
  "images": [
    { "file": "..._img1.jpg", "source_url": "https://..." }
  ]
}
```

Поле `title` имеет стандартный формат: `Тип \`БРЕНД\` Линейка объём`, что позволяет надёжно извлекать бренд regex-ом.

### Форматы поля composition

| Формат | Доля |
|---|---|
| Запятая (стандарт INCI) | 96.6% |
| Bullet `•` (Garnier и др.) | 1.5% |
| Точка с запятой | 1.8% |
| Плейсхолдер (`"-"`, `"Не найден"`) | ~6.5% |

## API

### Запуск

```bash
cd cvcosmetic
docker compose up --build
```

Первый запуск скачивает образ и модели EasyOCR (~2.6 GB итоговый образ).

### Эндпоинты

| Метод | URL | Описание |
|---|---|---|
| `GET` | `/health` | Проверка работоспособности |
| `POST` | `/api/recognition/label` | Распознавание этикетки |
| `GET` | `/docs` | Swagger UI |

### POST /api/recognition/label

**Тело запроса** — `multipart/form-data`:

| Поле | Тип | Обязательно | Описание |
|---|---|---|---|
| `image` | file | ✅ | JPG, PNG или WEBP |
| `title` | string | ❌ | Название продукта (повышает качество) |

**Пример ответа:**

```json
{
  "productCandidate": {
    "brand": "ЧИСТАЯ ЛИНИЯ",
    "product_type": "Крем",
    "product_name": "Аква-крем для лица ИДЕАЛЬНАЯ КОЖА Мгновенная матовость 50 мл",
    "volume": "50 мл",
    "barcode": null
  },
  "ingredientsRaw": "Aqua, Glycerin, Panthenol, Zinc Sulfate, Limonene.",
  "ingredientsParsed": ["Aqua", "Glycerin", "Panthenol", "Zinc Sulfate", "Limonene"],
  "ocrText": "...",
  "errors": [],
  "warnings": ["Состав/Ingredients не найден на изображении."]
}
```

### Пример вызова (Python)

```python
import requests

with open("label.jpg", "rb") as f:
    resp = requests.post(
        "http://localhost:8000/api/recognition/label",
        files={"image": ("label.jpg", f, "image/jpeg")},
        data={"title": "Крем `GARNIER` увлажняющий 50 мл"},
    )
print(resp.json())
```

## Что умеет система

### Извлечение сущностей из названия

Работает надёжно (данные из тестирования на 100 продуктах):

| Сущность | Точность |
|---|---|
| Бренд | 95.9% |
| Тип продукта | 85.3% |
| Объём | 87.1% |
| Мультиупаковка (`6x2 мл` → `2 мл`) | ✅ |
| Штрихкод EAN-8/EAN-13 (с валидацией) | зависит от изображения |

Поддерживаемые типы продуктов: Крем, Сыворотка, Тоник, Лосьон, Масло, Флюид, Ампула, Бальзам, Мусс, Гель, Пенка, Маска, Скраб, Шампунь, Кондиционер, Патчи, Тушь, Тени, Пудра и др.

### OCR

- Движок: **EasyOCR 1.7.2**, языки `ru + en`
- Препроцессинг: grayscale → autocontrast → sharpen → upscale если < 800px
- Хорошо читает: крупный латинский текст (названия продуктов, бренды)
- Хуже читает: мелкий текст состава, глянцевые поверхности, кириллицу под углом

### Парсинг состава

- Разделители: `,` `;` `•`
- Удаляет русские пояснения в скобках: `Zinc Sulfate (сульфат цинка)` → `Zinc Sulfate`
- Удаляет маркеры: `Состав:`, `Ingredients:`, `Состав/Ingredients:`
- Удаляет `*` в конце ингредиентов
- Среднее кол-во ингредиентов в датасете: **26.8**

## Ограничения и что стоит доделать

### Главное ограничение: OCR и изображения

**Проблема:** оба датасета содержат преимущественно маркетинговые фронтальные фото упаковки. Состав INCI напечатан на оборотной стороне мелким шрифтом и в существующих изображениях нечитаем.

Результат тестирования OCR на 30 backup-изображениях:
- Состав найден: **2 из 30 (6.7%)**
- В успешных случаях (ARTDECO, BLISTEX) — читается частично

**Решение:** для продакшн-сценария нужны фотографии оборотной этикетки крупным планом, где текст состава занимает большую часть кадра.

### Технический долг

| Задача | Приоритет | Сложность |
|---|---|---|
| Бренды `DR. ALTHEA`, `CLEAN+`, `LOVE IS…` не парсятся (спецсимволы `.` `+` `…`) | Высокий | Малая |
| `healthcheck` в Docker: нет `curl` внутри образа | Средний | Малая |
| 47 пустых папок в backup датасете | Средний | Краулинг |
| 49 продуктов с `composition="-"` | Средний | Краулинг |
| Препроцессинг: adaptive threshold для мелкого текста | Высокий | Средняя |
| Поиск продукта по штрихкоду в индексе датасета | Высокий | Средняя |
| Батч-эндпоинт `POST /api/recognition/batch` | Низкий | Средняя |
| Кэш результатов по hash изображения (БД) | Низкий | Большая |
